## 最初に入れるとよいこと

1. どんな課題や困りごとに向き合うのか
2. その課題に対して、どんな解決策を出したいのか
3. このシステムの目的は何か
4. 使う人は誰か
5. 画面で具体的に何ができるのか
6. 最初は作らなくてよいものは何か

自分の経験や、動かし方まで教えてほしいことも、最初のうちはプロンプトにそのまま書いて大丈夫です。

## 書き換える場所

- 課題を自分の生活や関心に合わせる
- 解決策を少し具体的にする
- 使う人や使う場面を足す
- 画面で起きる動きを足す
- 見た目の雰囲気を足す

<div class="callout" markdown="1">
  まだ全部が見えていなくても大丈夫です。書けるところまで書いて一度作り、触ってから直していけば十分です。
</div>
