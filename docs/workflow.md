# どの制作でも共通する作り方

このページは、Codexで何かを作るときの基本の流れです。

事例ページのプロンプトを使ってもよいですし、自分で作りたいものをそのまま書いても構いません。大事なのは、最初から完璧に頼むことではなく、作って、動かして、触って、直すことです。

## 1. 作業フォルダを用意する

作業フォルダは、今日作るものを入れておく箱です。

作るものごとに、別のフォルダを作ります。名前は英数字にしておくと、あとでトラブルが少なくなります。

例:

```text
my-story-app
today-photo
personality-match
```

## 2. `AGENTS.md` をそのフォルダに入れる

`AGENTS.md` は、Codexに「このフォルダではこう進めてほしい」と伝えるためのメモです。

ダウンロードしただけでは使われません。いま作るものの作業フォルダに入れてください。

別のサービスを作るために新しい作業フォルダを作ったら、そのフォルダにも `AGENTS.md` を入れます。

## 3. Codexでそのフォルダを選ぶ

Codexを開くと、チャットを書く画面があります。

画面の下の方に「プロジェクトで作業」のような場所があります。そこから、いま作るものの作業フォルダを選びます。

フォルダが見つからない場合は、「新しいプロジェクトを追加」からフォルダを追加します。

## 4. チャットに作りたいものを書いて送る

チャット欄に、作りたいものを書いて送ります。

事例ページを使う場合は、事例ページのプロンプトをコピーして、自分向けに少し書き換えてから貼ります。事例を使わない場合は、自分の言葉で書いて大丈夫です。

`AGENTS.md` に自分の前提や説明してほしい細かさは書いてあるので、ここでは作りたいものの中身を書きます。

できるだけ書くとよいこと:

1. どんな課題や困りごとに向き合うのか
2. その課題に対して、どんな解決策を出したいのか
3. このシステムの目的は何か
4. 使う人は誰か
5. 画面で具体的に何ができるのか
6. 最初は作らなくてよいものは何か

詳しく書けば書くほど、意図に近いものができやすくなります。ただし、まだ見えていない部分があっても大丈夫です。書けるところまで書いて一度作ってもらい、触ってから直していけば十分です。

```text
作りたいもの:
ここに書く

アプローチしたい課題:
ここに書く

解決策のアイデア:
ここに書く

このシステムの目的:
ここに書く

使う人:
ここに書く

具体的な動き:
- ここに書く
- ここに書く
- ここに書く

最初は作らなくてよいもの:
- ここに書く

まずは、PC上で動く小さな形で作ってください。
```

## 5. 実際に動くか試す

Codexが作り終わったら、まず動くか試します。

Codexが「このコマンドを実行してください」「このURLを開いてください」と書いてくれた場合は、その通りに進めます。

どこを押せばよいか、どこにコマンドを貼ればよいか、どのURLを開けばよいかが分からない場合は、すぐに聞いてください。

```text
今作ったものを自分のPCで動かして確認したいです。
私は初心者なので、次に何をすればよいかを1手順ずつ書いてください。
コマンドが必要なら、どこに貼るのかも説明してください。
ブラウザで開くものならURLを、そうでないものなら起動方法や確認方法を教えてください。
```

## 6. 実際に使ってみる

動かせたら、少し触ってみます。

今回はWebサービスから始めるので、ブラウザで開いて確認することが多いです。ただし、作るものがいつもブラウザで開くものとは限りません。スマートフォンアプリならシミュレータや実機で開くかもしれませんし、カメラを使うものなら権限を許可して試すかもしれません。

見るところ:

- 最初に何をすればよいか分かるか
- ボタンや入力欄を触れるか
- 文字が読みにくくないか
- 自分が作りたいものに近いか
- スマートフォンで使うなら、スマートフォンでも見やすそうか

## 7. 気になるところを伝えて直す

最初にできたものは完成品ではありません。できたものを下敷きにして、とにかく直していきます。

一度に全部直そうとせず、まず気になるところを1つだけ選びます。

```text
今作ったものを見ました。

気になるところ:
ここに書く

残したいところ:
ここに書く

大きく作り替えすぎず、今の形を活かして直してください。
修正後に、どこを変えたかと、どう確認するかを教えてください。
```

## 8. この流れをくり返す

Codexで作るときは、1回の指示で終わりではありません。

1. 作る
2. 動かす
3. 触る
4. 気になるところを伝える
5. 直す

このくり返しで、だんだん自分の作りたいものに近づけます。

## 迷ったら、この流れで案内してもらう

どこから進めればよいか分からなくなったら、Codexにこのまま頼みます。

```text
このページの「共通の制作フロー」に沿って、1つずつ一緒に進めてください。
今、私はどの段階にいるのかを確認してから、次にやることを1つだけ教えてください。
操作が必要な場合は、どこを押すのか、どこに入力するのかも説明してください。
```
