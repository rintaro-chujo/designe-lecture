# 学生配布用 AGENTS.md テンプレート

このファイルは、学生が自分の作業フォルダに `AGENTS.md` という名前で置くための原稿です。
`AGENTS.md` はフォルダごとに置きます。別のサービスを作るために新しい作業フォルダを作ったら、そのフォルダにも `AGENTS.md` を入れます。
Web教材では、この内容を学生が少し編集してダウンロードできるようにします。

```markdown
# 私のCodex作業メモ

この `AGENTS.md` は、このファイルが入っている作業フォルダ用のメモです。
別のサービスを作る新しいフォルダでは、そのフォルダにも `AGENTS.md` を入れてください。

## 私の前提

私は商学部3年生です。
プログラミングは初心者です。
専門用語が出てきたら、短く日本語で説明してください。

## 進め方

- まず、PC上で動く最小限のものを作ってください。
- そのあと、画面を見ながら少しずつ直していきたいです。
- 一度に大きく変えすぎず、変更した内容を短く説明してください。
- コマンドが必要な場合は、どこに貼るのかも説明してください。
- 作ったあとは、ブラウザで開く方法を必ず教えてください。

## 私に確認してほしいこと

次の場合は、勝手に進めずに確認してください。

- いま作っているものの目的が変わりそうなとき
- ファイルを大きく作り替えるとき
- 個人情報を画面に出す可能性があるとき
- 追加のアカウント作成や外部サービス利用が必要なとき

## エラーが出たとき

エラーが出たら、次の順番で進めてください。

1. 何が起きているかを初心者向けに説明する
2. まず試す修正を1つ選ぶ
3. 修正する
4. もう一度、動かし方を教える

## デザインの好み

読みやすく、落ち着いた画面にしたいです。
文字が小さすぎたり、情報が詰まりすぎたりしないようにしてください。
ただし、何も印象に残らない一般的な画面にはしたくありません。

## 宿題の目標

2週間後までに、自分または誰かが実際に使える小さなサービスを1つ作ります。
デプロイはまだ必須ではありません。
自分のPCで動き、使い方を説明できる状態を目指します。

## 最後に毎回教えてほしいこと

作業が終わったら、次の3つを教えてください。

1. 何を変更したか
2. どうやって動かして確認するか
3. 次に改善できること
```

## 編集候補

学生が慣れてきたら、次の項目を自分で足してよいです。

- 自分のサービスの目的
- 読んでほしい相手
- デザインの好み
- 使いたい色
- 苦手な説明のされ方
- Codexに勝手にやってほしいこと
- Codexに毎回確認してほしいこと
